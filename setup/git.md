# Git Setup Guide

## What Is Git?

Git is a version control system.

Version control means keeping track of changes over time.

Think of Git as:

A time machine for your projects.

---

## Why We Use Git

Benefits:

* Track progress
* Undo mistakes
* Collaborate safely
* Maintain project history

---

## Step 1: Install Git

Download from:

https://git-scm.com

Install using default settings.

---

## Step 2: Verify Installation

Open Terminal.

Run:

git --version

You should see a version number.

---

## Step 3: Configure Git

Replace the example information with your own.

git config --global user.name "Your Name"

git config --global user.email "[your@email.com](mailto:your@email.com)"

---

## Basic Workflow

### Check Status

git status

Shows modified files.

---

### Add Changes

git add .

Stages changes.

---

### Create Commit

git commit -m "Describe your changes"

Creates a save point.

---

### Upload Changes

git push

Sends updates to GitHub.

---

## Typical Session

1. Make changes.
2. Save files.
3. git status
4. git add .
5. git commit
6. git push

---

## Important Rule

Commit often.

Small commits are easier to understand and recover.

---
